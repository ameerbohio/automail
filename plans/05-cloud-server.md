# Cloud Server

**Language**: Go  
**Tag**: [CORE]  
**Role**: Stateless, zero-knowledge job router. Stores and forwards ciphertext; never decrypts.

---

## Responsibilities

1. Authenticate senders (JWT)
2. Serve the printer public key directory
3. Generate pre-signed MinIO upload URLs
4. Accept job submissions (ciphertext metadata only)
5. Accept persistent mTLS WebSocket connections dialed by printers; hold a per-node connection registry
6. Attempt immediate dispatch; enqueue if blocked
7. Cache printer state from socket frames; publish availability events
8. Run dispatcher goroutine: dequeue blocked jobs when printers become available
9. Relay job status events via SSE to the sender's browser
10. Trigger MinIO blob deletion after confirmed delivery
11. Write immutable audit log entries

## Request Handlers

### `GET /healthz`

No auth. Returns 200 if Postgres and Redis are reachable. Used by Traefik health checks.

### `POST /auth/login`

Accepts `{ email, password }`. Verifies bcrypt hash. Returns access JWT (RS256, 15 min) and sets refresh token cookie (httpOnly, Secure, SameSite=Strict, 7 days).

### `POST /auth/refresh`

Reads refresh token from cookie. Validates against `refresh_tokens` table (hash comparison). Issues new access JWT and rotates refresh token (old token invalidated, new one set).

### `GET /mailboxes/:id/public-key`

JWT-authenticated. Returns `{ public_key_pem: "..." }` from `mailboxes.public_key_pem`. The cloud server does not use this key for anything — it is a key directory only.

### `POST /jobs/upload-url`

JWT-authenticated. Generates a MinIO pre-signed PUT URL for a new object:

```go
presignedURL, err := minioClient.PresignedPutObject(ctx, "automail", objectKey, 15*time.Minute)
```

Returns `{ upload_url, blob_ref }`. The browser uploads the ciphertext blob directly to MinIO. The cloud server never receives the blob.

### `POST /jobs`

JWT-authenticated. Request body:

```json
{
  "encrypted_key": "base64",
  "blob_ref": "blobs/<uuid>",
  "slot_id": "uuid",
  "page_count": 3
}
```

Flow:
1. Validate: slot exists, printer registered for slot's building
2. Insert job with status `submitted`
3. Write audit event `job_submitted`
4. Attempt immediate dispatch (see Dispatch section below)
5. Return `{ job_id, status }`

### `GET /jobs/:id/stream`

JWT-authenticated; sender must own the job.

Server-Sent Events stream. Subscribe to Redis pub/sub channel `job:<id>:status`, relay messages to the response writer:

```go
w.Header().Set("Content-Type", "text/event-stream")
w.Header().Set("Cache-Control", "no-cache")
w.Header().Set("Connection", "keep-alive")
flusher := w.(http.Flusher)

pubsub := rdb.Subscribe(ctx, "job:"+jobID+":status")
for msg := range pubsub.Channel() {
    fmt.Fprintf(w, "data: %s\n\n", msg.Payload)
    flusher.Flush()
}
```

Works across nodes because all nodes publish to the same Redis channel.

### `GET /internal/printer-link` (mTLS WebSocket upgrade)

The printer dials this endpoint and holds the connection open. The handler verifies the client cert against the internal CA, upgrades to a WebSocket, and runs the link loop on the **owner node** (the node that accepted this socket):

```go
func (h *Hub) ServePrinterLink(w http.ResponseWriter, r *http.Request) {
    conn := upgrade(w, r)                     // mTLS already verified by tls.Config
    reg := readRegisterFrame(conn)            // { mailbox_id, slot_occupancy }

    h.registry.Add(reg.MailboxID, conn)       // in-process soft state, this node only
    defer h.registry.Remove(reg.MailboxID, conn)

    // This node now owns the socket: subscribe so any node can route dispatches here
    sub := rdb.Subscribe(ctx, "mailbox:"+reg.MailboxID+":dispatch")
    go h.pumpDispatch(conn, sub)              // Redis msg → write dispatch frame to socket

    cacheState(reg.MailboxID, reg.toState())  // seed Redis mailbox:<id>:state

    for {                                     // read loop: state + status frames
        frame := readFrame(conn)              // returns on error → defer cleans up
        switch frame.Type {
        case "state":  onState(reg.MailboxID, frame)   // refresh cache; publish availability
        case "status": onStatus(frame)                 // see status handling below
        }
    }
}
```

**State frames** refresh the printer cache and signal availability (formerly the heartbeat POST):

```go
rdb.Set(ctx, "mailbox:"+mailboxID+":state", json(frame), 90*time.Second)
if frame.Status == "idle" {
    rdb.Publish(ctx, "mailbox:"+mailboxID+":available", "1")
}
```

A closed socket (read error) removes the connection from the registry; the `mailbox:<id>:state` key then TTL-expires, which is the offline signal. WebSocket ping/pong keepalive runs alongside the read loop.

**Status frames** carry `{ job_id, status: "printing" | "delivered" | "failed" }` and are handled exactly as the old status callback did:

```go
// Update job status in Postgres; write audit event
// Publish to Redis pub/sub for SSE relay
rdb.Publish(ctx, "job:"+jobID+":status", json({ status })) // job_id omitted: the channel name already scopes it

// On "delivered": request MinIO blob deletion
if frame.Status == "delivered" {
    minioClient.RemoveObject(ctx, "automail", job.BlobRef)
    // Update job blob_deleted_at
}
```

## Dispatch Logic

### Immediate Dispatch (called from `POST /jobs` handler)

```go
func tryDispatch(ctx context.Context, job Job) (dispatched bool) {
    state := getPrinterState(ctx, job.MailboxID)  // from Redis
    if state.Status != "idle" { return false }
    slot := state.SlotOccupancy[job.SlotID]
    if slot.Current >= slot.Max { return false }

    // Atomic claim in Postgres
    ok := claimJob(ctx, job.ID)  // SELECT FOR UPDATE NOWAIT, UPDATE status='dispatching'
    if !ok { return false }

    // Generate pre-signed MinIO read URL (5 min)
    readURL := minioPresignedGet(ctx, job.BlobRef, 5*time.Minute)

    // Route the dispatch to whichever node owns the printer's socket.
    // PUBLISH instead of a direct call: the dispatching node may not be
    // the node holding the connection.
    payload := json({ job_id: job.ID, encrypted_key: job.EncryptedKey, blob_url: readURL })
    receivers, _ := rdb.Publish(ctx, "mailbox:"+job.MailboxID+":dispatch", payload).Result()
    if receivers == 0 {
        // No node has a live socket for this printer → it is offline.
        revertJobStatus(ctx, job.ID, "queued")
        return false
    }
    writeAuditEvent(ctx, job.ID, "job_dispatched")
    return true
}
```

If immediate dispatch fails (printer not idle, slot full, or no live socket): add to Redis Stream via `XADD`.

### Why publish instead of call

The printer holds **one** persistent socket, terminated on a single node (the *owner node*). But `tryDispatch` may run on any node — the dispatcher goroutine runs on all of them. A node cannot write to a socket it does not hold, so dispatch is routed through Redis pub/sub: the owner node subscribed to `mailbox:<id>:dispatch` when it accepted the socket, and `pumpDispatch` writes the received message down the wire.

This is the mirror image of SSE status fan-out (which routes a status update to whichever node holds the sender's SSE connection). Both use the same primitive — Redis pub/sub as the cross-node bridge — so the "all coordination goes through Redis" contract holds even though a live socket is in-process state. See [03-scaling.md](03-scaling.md) "Printer Connection Fan-In".

### Dispatcher Goroutine

One goroutine per node. Subscribes to `mailbox:<id>:available` channels via Redis pub/sub. On each event, runs `XREADGROUP` on `jobs:pending` stream to pick up blocked jobs and re-attempt dispatch (same `tryDispatch` logic).

See [03-scaling.md](03-scaling.md) for the full Redis Streams + consumer group implementation.

## Zero-Knowledge Guarantee

The cloud server code must never:
- Call `crypto.subtle` or any decryption function on `encrypted_key`
- Log the value of `encrypted_key`
- Pass `encrypted_key` to any endpoint other than the printer microservice

`encrypted_key` is stored as `BYTEA` in Postgres and passed as-is to the printer. The cloud server treats it as an opaque blob.
